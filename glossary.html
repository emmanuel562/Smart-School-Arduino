Glossary

Words every engineer knows

Every term you've encountered in class — defined clearly, with examples. This list grows after every session.

🔍
A
Actuator
Session 2
Any device that takes electricity from the Arduino and converts it into something physical — light, sound, or movement.
Examples: LED (light), buzzer (sound), servo motor (movement), DC motor (wheels spinning).
Analog
Session 1 & 2
A signal or pin that can hold any value within a range — not just ON or OFF. On the Arduino, analog output values go from 0 to 255, and analog input values go from 0 to 1023.
Like a dimmer switch — you control exactly how much light comes out, not just on or off.
analogWrite(pin, 128) // half power
analogWrite()
Session 2
A function that sends a PWM signal to a pin — controlling the power level from 0 (off) to 255 (full power). Only works on PWM pins marked with ~ on the board.
analogWrite(9, 200) // 78% power on pin 9
Arduino
Session 1
An open-source electronics platform consisting of a microcontroller board and a programming environment. The processing unit in our Input → Processing → Output model.
The Arduino Uno R3 is the specific board we use in this programme.
B
Breadboard
Session 2
A reusable board for building circuits without soldering. Components and wires push into holes that are connected in rows underneath. The two outer rails (marked + and –) are for power and ground.
Think of it as a temporary construction site for your circuit.
Buzzer (Piezo)
Session 2
An output device that converts electricity into sound by vibrating a small disc rapidly. The frequency of vibration determines the pitch — higher frequency means higher pitch.
tone(8, 440) // play A4 note (440 Hz)
C
C++
Session 1
The programming language used to write Arduino code. One of the most widely used languages in the world — used in game engines, operating systems, robotics, and embedded systems.
The code you write in the Arduino IDE is C++ with some Arduino-specific additions.
Comment
Session 1
A line of code that the Arduino ignores completely. Used to explain what your code does for anyone reading it — including future you.
// This is a comment — the board skips this line
D
delay()
Session 1
A function that pauses the program for a set number of milliseconds. 1000 milliseconds = 1 second. The board does absolutely nothing during a delay.
delay(500) // wait half a second
Digital
Session 1 & 2
A signal or pin that has exactly two possible states: HIGH (on, 1) or LOW (off, 0). Nothing in between. Like a light switch — fully on or fully off.
Digital pins on the Arduino Uno: 0 through 13.
digitalWrite()
Session 1
A function that sets a digital pin to HIGH (send electricity out) or LOW (stop electricity). Used to turn LEDs, buzzers, and other digital outputs on or off.
digitalWrite(13, HIGH) // turn pin 13 ON
F
Frequency
Session 2
The number of vibrations per second, measured in Hertz (Hz). For a buzzer, frequency controls pitch — the higher the number, the higher the sound. Middle C on a piano is 262 Hz.
262 Hz = C4 · 440 Hz = A4 · 4000 Hz = very sharp high beep
Function
Session 1
A named, reusable block of code that performs a specific job. You call a function by writing its name followed by parentheses. The Arduino IDE requires two functions in every sketch: setup() and loop().
digitalWrite(pin, HIGH) // calling the digitalWrite function
G
GND (Ground)
Session 1
The reference point for all voltage measurements in a circuit — the return path for electricity. Every component needs a path back to GND for current to flow. Think of it as the negative terminal of a battery.
H
HIGH
Session 1
A constant that represents the "on" state for a digital pin — 5V on the Arduino Uno. When you write digitalWrite(pin, HIGH), the pin sends out 5 volts of electricity.
HIGH = 1 = ON = 5V
I
#include
Session 2
A directive at the top of your code that tells the Arduino to load a library before compiling. It gives your sketch access to extra functions someone else already wrote.
#include <Servo.h> // load the Servo library
Input
Session 1
The first step in the Input → Processing → Output model. Inputs collect information from the environment and send it to the Arduino. Sensors, buttons, and microphones are all inputs.
int
Session 1
Short for "integer" — a data type for whole numbers (no decimals). Used to declare variables that store whole number values like pin numbers, delay times, or brightness levels.
int ledPin = 7; // ledPin holds the value 7
L
LED
Session 1 & 2
Light Emitting Diode. A component that converts electricity into light. Has two legs — the longer leg (anode, +) connects to power, the shorter leg (cathode, –) connects to GND. Always needs a resistor to limit current.
Library
Session 2
A collection of pre-written code that adds extra functions to your sketch. Instead of writing all the servo control code yourself, you import the Servo library and use its ready-made functions.
Engineers always build on top of what others have already made. Libraries are how that happens.
loop()
Session 1
One of the two required functions in every Arduino sketch. Everything inside loop() runs continuously, over and over, for as long as the board has power. Your main program behaviour goes here.
LOW
Session 1
A constant that represents the "off" state for a digital pin — 0V. When you write digitalWrite(pin, LOW), the pin stops sending electricity.
LOW = 0 = OFF = 0V
M
Microcontroller
Session 1
A small computer on a single chip that contains a processor, memory, and input/output pins. The brain of the Arduino board — it stores and runs your code.
The large black rectangular chip in the middle of the Arduino Uno board is the ATmega328P microcontroller.
Motor
Session 2
A device that converts electrical energy into rotational movement. DC motors spin continuously (used in wheels). Servo motors rotate to a precise angle (used in robot arms). Stepper motors move in precise steps (used in 3D printers).
N
noTone()
Session 2
A function that stops any sound playing through a buzzer on a specified pin.
noTone(8) // stop buzzer on pin 8
O
Output
Session 1
The third step in the Input → Processing → Output model. Outputs are what the Arduino does after processing — something physical you can see, hear, or feel. LEDs, buzzers, servos, and screens are all outputs.
P
Pin
Session 1
A connection point on the Arduino board where you attach components using wires. Pins can be digital (HIGH/LOW), analog (0–1023 range), or power supply (5V, 3.3V, GND).
pinMode()
Session 1
A function used in setup() to declare whether a pin will be used as an INPUT (receiving data) or OUTPUT (sending data). You must call this before using the pin.
pinMode(7, OUTPUT) // pin 7 will send electricity out
Processing
Session 1
The middle step in the Input → Processing → Output model — where the Arduino board reads your code and decides what to do with incoming information. The Arduino board itself is the processing unit.
PWM
Session 2
Pulse Width Modulation. A technique for simulating analog output using a digital pin — by switching the pin on and off very rapidly, the average power is controlled. This is how analogWrite() works on the Arduino. PWM pins are marked with ~ on the board.
PWM pins on the Uno: 3, 5, 6, 9, 10, 11
R
Resistor
Session 2
A component that limits the flow of electrical current. Used with LEDs to protect them from receiving too much current and burning out. Resistance is measured in Ohms (Ω). We use 220Ω resistors with LEDs in this programme.
A resistor is like a narrow section in a pipe — it slows down the flow.
RGB LED
Session 2
A single LED component that contains three separate LEDs — Red, Green, and Blue. By controlling the brightness of each (0–255 using analogWrite), you can produce any of 16 million possible colours.
analogWrite(9, 255) // full red
S
Sensor
Session 1
A device that detects something in the physical environment and converts it to an electrical signal. Sensors are inputs — they feed information to the Arduino for processing.
Sound sensor (detects noise), light sensor (detects brightness), motion sensor (detects movement), temperature sensor (detects heat).
Servo Motor
Session 2
A motor you control by angle rather than speed. You command it to go to a specific position between 0° and 180°, and it holds that position. Used in robot arms, steering systems, and door mechanisms.
myServo.write(90) // move to 90 degrees
setup()
Session 1
One of the two required functions in every Arduino sketch. Code inside setup() runs exactly once — when the board powers on or resets. Use it to declare pin modes and start serial communication.
Sketch
Session 1
The Arduino word for a program. When you write code in the Arduino IDE and save it, that file is called a sketch. Every sketch must have both a setup() and a loop() function.
T
tone()
Session 2
A function that generates a square wave at a given frequency on a pin, causing a piezo buzzer to produce sound. The sound continues until noTone() is called or a duration is specified.
tone(8, 440, 500) // play A4 for 500ms on pin 8
V
Variable
Session 1
A named storage location that holds a value you can use and change throughout your program. Declaring a variable at the top makes your code easier to read and modify — change the value once, it updates everywhere it's used.
int delayTime = 1000; // change this one number to change speed everywhere
void
Session 1
A keyword placed before a function definition that means "this function doesn't return a value — it just does something." Both setup() and loop() are void functions. Don't worry too much about this — just know it's required.
void loop() { // "this function does stuff but gives nothing back" }
W
Wokwi
Session 1
A free, browser-based Arduino simulator at wokwi.com. Build virtual circuits and test code without any physical hardware. Useful for practising between sessions or when you don't have access to a board.
