The Big IdeaInput → Processing → Output

Every electronic device in the world follows this same pattern. From your phone to a robot to a traffic light — they all do this:

👂

INPUT

Collects information from the world

→
⚙️

PROCESSING

Decides what to do with the information

→
📢

OUTPUT

Does something you can see, hear, or feel

🤔 Example: When you clap near a clap-sensor light — the clap is the INPUT, the Arduino PROCESSES it, the light turning on is the OUTPUT.

👂

Step 1Inputs & Sensors

Sensors are how the Arduino hears the world. They collect data from the environment and send it to the board. Even a coat on a sensor is an input — anything that sends a signal counts!

Step 2Processing — The Arduino Board

The Arduino board is the brain. It reads what sensors tell it, runs your code, and decides what to do. But remember —

⚠️ The robot is dumb until YOU tell it what to do.
Without your code, the Arduino does nothing. Your instructions are what make it smart.
Arduino Uno R3 — Parts You Must Know
🔌

USB Port

This is how you connect the Arduino to your computer. Your code travels through here. It also gives the board power.

D

Digital Pins (0–13)

These work with ON or OFF signals only. Pin 13 is connected to the small LED on the board. Use these for LEDs, buzzers, and buttons.

A

Analog Pins (A0–A5)

These can read values in between — not just on or off. Used for sensors that measure things like light level or temperature (many different values).

⚡

Power Pins (5V, 3.3V, GND)

5V and 3.3V give power to your components. GND (Ground) is the return path — electricity always needs somewhere to go back to.

Digital vs Analog — What's the difference?
🔵 DIGITAL PIN — Only 2 possible values
0
1
LOW = OFF HIGH = ON
Nothing in between. It's either OFF or ON. That's it.
🟣 ANALOG PIN — 256 possible values (0 to 255)
0
128
255
LOW (0) = Off / Dark / Quiet HIGH (255) = Bright / Loud
Everything in between too — like a dimmer switch, not just on/off.

The Code You WroteBlink — Your First Program

This is the code that made your LED blink. Read it again — you'll understand more now.

blink.ino
// Tell the Arduino to set up the LED pin
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);  // pin 13 → sending out
}

// This runs forever — over and over
void loop() {
  digitalWrite(LED_BUILTIN, HIGH);  // LED ON  (1)
  delay(1000);                   // wait 1 second
  digitalWrite(LED_BUILTIN, LOW);   // LED OFF (0)
  delay(1000);                   // wait 1 second
}
void setup()
Runs once when the board turns on. Like loading a game — it sets everything up before you start playing.
pinMode(pin, OUTPUT)
Tells a pin its job. OUTPUT means it will be sending electricity out (like pin 13 to the LED).
void loop()
Runs forever, over and over, until you turn the board off. Your main behaviour lives here.
digitalWrite(pin, HIGH)
Sends electricity to a pin. HIGH = 1 = ON. The LED turns on.
digitalWrite(pin, LOW)
Stops electricity to a pin. LOW = 0 = OFF. The LED turns off.
delay(1000)
Tells the board to wait. 1000 = 1 second. 500 = half a second. Change this number to change the speed.

5 seconds

✅ B — delay() counts in milliseconds. 1000ms = 1 second, so 500ms = half a second.

🧠 The 5 Things to Remember

1️⃣ Every device: Input → Processing → Output
2️⃣ Sensors are inputs. LEDs, motors, speakers are outputs.
3️⃣ void setup() runs once. void loop() runs forever.
4️⃣ HIGH = ON (1). LOW = OFF (0).
5️⃣ The Arduino is dumb until YOU give it instructions.
