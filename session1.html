2 Inputs & Sensors

Sensors are how the Arduino hears the world. They collect information from the environment and send it to the board. Even your written code counts as an input — it's the set of instructions the Arduino follows.

🎤
Sound Detector
Like a microphone. Detects sound or clapping.
☀️
Light Detector
Detects how bright or dark the environment is.
🔘
Button / Switch
Pressed = signal sent. Not pressed = no signal.
🏃
Motion Sensor
Detects when something is moving nearby.
💻
Code (Instructions)
Your written code tells the Arduino what to watch for and when to act.

3 The Arduino Board

The Arduino Uno R3 is your processing unit. It connects to your computer via USB, receives your code, and runs it forever. But remember —

⚠️
The Arduino is dumb until YOU give it instructions. Without your code, it does nothing. Your instructions are what make it smart.
🔌

USB Port

Connects the Arduino to your computer. Your code travels through here. It also powers the board during development.

D

Digital Pins (0 – 13)

Work with ON or OFF signals only. Pin 13 is connected to the small built-in LED on the board. Use these for LEDs, buzzers, and buttons.

A

Analog Pins (A0 – A5)

Can read values between 0 and 1023 — not just on or off. Used for sensors that measure things like light level or temperature.

⚡

Power Pins (5V, 3.3V, GND)

5V and 3.3V supply power to components. GND (Ground) is the return path — electricity always needs somewhere to flow back to.

4 Outputs

Outputs are how the Arduino acts on the world. After processing your code, something physical happens — something you can see, hear, or feel.

💡
LED
Converts electricity into light. Digital (on/off) or analog (brightness).
🔊
Speaker / Buzzer
Converts electricity into sound. Frequency controls the pitch.
📺
LCD Screen
Displays text or numbers. Used to show sensor readings.
⚙️
Motor
Converts electricity into movement — wheels, arms, fans.

5 Your First Program — Blink

This is the code that made your LED blink. Read it again — you'll understand more now than you did when you first saw it.

blink.ino 
// Tell the Arduino to set up the LED pin
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);  // pin 13 → sending electricity out
}

// This runs forever — over and over
void loop() {
  digitalWrite(LED_BUILTIN, HIGH);  // LED ON  (1)
  delay(1000);                       // wait 1 second
  digitalWrite(LED_BUILTIN, LOW);   // LED OFF (0)
  delay(1000);                       // wait 1 second
}
void setup()
Runs once when the board turns on. Like loading a game — sets everything up before the main action starts.
pinMode(pin, OUTPUT)
Tells a pin its job. OUTPUT means it will be sending electricity out. You must call this in setup() before using the pin.
void loop()
Runs forever, over and over, until you turn the board off. Your main behaviour lives here.
digitalWrite(pin, HIGH)
Sends electricity to a pin. HIGH = 1 = ON. The LED turns on.
digitalWrite(pin, LOW)
Stops electricity to a pin. LOW = 0 = OFF. The LED turns off.
delay(1000)
Tells the board to wait. 1000 = 1 second. 500 = half a second. Change this number to change the blink speed.
